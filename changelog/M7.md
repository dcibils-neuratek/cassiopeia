# Milestone 7 — Timers, delays & SLAs

Shipped 2026-07-01 · commit `cd57551`

Give the engine a sense of time.

## Functional
- New **Timer node** ("Wait"): park a run for a **fixed delay** (seconds) or
  **until a date** read from the context (`untilPath`). A background **scheduler**
  resumes the run automatically once the time passes.
- The **Portal** shows "⏱ Waiting until …" and polls so it resumes on its own;
  **Monitor** shows a sleeping instance's wake time.
- **User-task SLA**: set "due within N hours"; the task gets a due date. Monitor
  flags **overdue** vs due tasks (and it feeds the M8 Inbox ordering).
- New audit events: `timer.scheduled`, `timer.fired`.

## Technical
- **model** — `TimerNode { delaySeconds?, untilPath? }` added to the `Node` union
  and `NodeType`; `UserTaskNode.slaHours?`.
- **engine** — timer case parks and returns a new `sleeping` `AdvanceResult`
  (instance stays `waiting`); `resumeTimer()` steps past the timer and advances;
  `validateDefinition` requires one outgoing edge and a delay-or-until.
- **db** — new `timers` table (`wake_at`, `status`) + `tasks.due_at` migration;
  `createTimer` / `dueTimers` / `fireTimerRow` / `openTimerForInstance`;
  `listOpenTasks`.
- **runtime** — a shared `onParked()` opens the right thing after every advance:
  a task (with the SLA due date) or a timer (computed `wakeAt` — `untilPath`
  wins over `delaySeconds` when it parses). `fireDueTimers()` resumes due timers;
  `startScheduler()` polls every second (`unref`ed).
- **server** — `startScheduler()` on boot; `/instances/:id` now returns
  `openTimer`.
- **web** — Designer timer node (palette / colour / render / inspector with
  seconds + until-path) and user-task SLA field; Portal sleeping state + polling;
  Monitor sleeping/overdue badges.

## Verified
- Live API: `start → sleeping` (status `waiting`, `openTimer.wakeAt` ~3s out);
  after the scheduler tick → `completed` with `timer.scheduled → timer.fired →
  instance.completed`. SLA task gets a `dueAt` ~60 min out.
- Browser: Designer timer palette + inspector; Portal parks ("Waiting until …")
  then auto-resumes to `completed` (full audit trail visible).

# Milestone 17 — Quality & delivery

Shipped 2026-07-20

Pay down the "no tests, no container" debt and make the app deployable.

## Functional / operational
- **Automated tests** for the riskiest pure logic — the **engine**, **expr**, and
  **form-kit** — runnable with `pnpm test` (20 tests, `node:test`, no new deps).
- **CI**: GitHub Actions runs **typecheck + tests** on every push and PR.
- **Docker**: `docker compose up --build` brings the whole app up in a container;
  SQLite data + the encryption key persist in a named volume.
- **Configuration via env**: `PORT`, `CASSIOPEIA_SECRET_KEY`, `API_TARGET`,
  documented in the README.
- **Postgres path documented**: all SQL is isolated in one repo module, so moving
  off `node:sqlite` is a contained change (not done here — it needs a live
  Postgres to verify, so it's the documented next step rather than shipped
  untested).

## Technical
- **tests** — `packages/expr/test` (precedence, booleans, dotted paths, missing
  ids), `packages/engine/test` (gateway routing, retry exhaust/recover, error
  routing, timer parking, multi-instance fan-out, subprocess merge,
  validateDefinition), `packages/form-kit/test` (required, visibleIf, computed
  derive, pages, visiblePatch). Added the root `test` script.
- **.github/workflows/ci.yml** — pnpm + Node 22 (needed for `node:sqlite`) →
  `pnpm install --frozen-lockfile` → `pnpm typecheck` → `pnpm test`.
- **Dockerfile / docker-compose.yml / .dockerignore** — layer-cached install, data
  volume, port exposure; `vite.config` binds `host: true` and honors `API_TARGET`
  so the web server is reachable inside a container.

## Verified
- `pnpm test` → **20 passing, 0 failing**; `pnpm typecheck` clean. CI mirrors both.

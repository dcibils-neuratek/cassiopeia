# Milestone 9 — Governance: versioning, import/export, data dictionary

Shipped 2026-07-01 · commit `52097ef`

Operate workflows over time. Surfaced in Build → **Manage** and Monitor.

## Functional
- **Version history** — list a process's published versions (with node/edge
  counts) and **restore** any of them into the working draft.
- **Export / Import** — download a portable **workflow bundle** (definition +
  every referenced form + connector, with **secrets stripped**) as JSON, and
  import a bundle into a workflow. Import **never clobbers** an existing
  connector (so live API keys are safe).
- **Data dictionary** — the context-key contract for a process: which keys are
  **produced** vs **consumed**, by which step, with **warnings** for keys that are
  read but never produced (likely typos or a missing step).
- **Audit CSV** — download an instance's full audit trail as CSV from Monitor.

## Technical
- **db** — `listDefinitionVersions(id)`.
- **governance.ts** (new) — `exportBundle` (strips `apiKey`/`token`/`secret`/…),
  `importBundle` (saves forms; adds only missing connectors; stores as a fresh
  draft), `dataDictionary` (walks forms/service maps/gateway conditions/timer
  paths; extracts dotted identifiers from expressions), `auditCsv` (RFC-escaped).
- **server** — `/definitions/:id/versions`, `/restore/:version`, `/export`,
  `/import`, `/data-dictionary`; `/instances/:id/audit.csv` (with
  content-disposition).
- **web** — Designer **Manage** modal (Version history / Data dictionary / Import
  tabs) + **Export** button (blob download); Monitor **⬇ CSV** link.

## Verified
- Live API: versions list (v1/v2), data dictionary (12 entries incl. 2 accurate
  warnings), export bundle with `apiKey` stripped, import as a copy, restore v1,
  CSV with correct escaping.
- Browser: Manage modal renders Version history (with Restore) and the Data
  dictionary (produced/consumed + warnings).

## Notes
- Because a service task without an explicit `outputMap` merges its whole result,
  the dictionary can't statically know keys like `riskScore` are produced — so it
  correctly flags them. Adding output maps (or documenting them) resolves the
  warning; this is the tool doing its job.

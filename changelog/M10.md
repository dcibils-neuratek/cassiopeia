# Milestone 10 — Analytics & AI process analyst

Shipped 2026-07-01 · commit `0332d7f`

Close the loop from execution back to design. Surfaced in **Stats**.

## Functional
- **Per-node cycle time** derived from the event log: average and max time spent
  at each step, with the **bottleneck** highlighted (🐢).
- **Average end-to-end completion time**, completed count, and **gateway branch
  distribution** (how often each path is taken) and **failures by node**.
- A process selector to scope the analytics to one workflow.
- **✦ Analyze with AI** — an **AI process analyst** reads the metrics and the
  process structure and returns concrete improvement suggestions (biggest
  bottleneck/risk, 3–5 actionable changes, data-quality concerns).

## Technical
- **analytics.ts** (new) — `computeAnalytics(defId)` walks each instance's events,
  measuring per-node dwell time between successive `node.entered` events (closing
  the last node at completion/failure/last-event), aggregating visits/avg/max,
  gateway `to`-distribution, `service.failed` counts, and completed cycle time.
  `analyzeProcess()` formats a compact metrics summary and calls the LLM.
- **describe.ts** — extracted a reusable `callDescriber(system, user, override)`
  helper (Claude Haiku default; any OpenAI-compatible provider) now shared by
  Describe and the analyst.
- **server** — `GET /analytics/:defId`; `POST /definitions/:id/analyze`.
- **web** — Stats gains a "Cycle time & AI analyst" section: process selector,
  avg/completed/bottleneck metric cards, a per-step timing table (bottleneck row
  highlighted), gateway-branch summary, and the AI suggestions panel.

## Verified
- Live API: `m7timer` → bottleneck = the `Cooldown` timer at ~3.5s (exact, = the
  3s delay + scheduler latency), start/end at 0ms; `m6fail` → failures-by-node = 2
  on "Flaky call"; the analyst pipeline returns suggestions (exercised via the
  mock LLM override).
- Browser: Stats section renders with the bottleneck highlighted and the metric
  cards populated (verified via DOM inspection at viewport coordinates).

## Notes
- The **AI analyst and ✦ Describe need a real `describer` API key** (set it in
  Settings). The pipeline was verified against the local mock LLM; suggestion
  quality depends on your configured model.

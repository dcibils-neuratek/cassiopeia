# Milestone 1 — Scaffold & walking-skeleton engine (M0/M1)

Baseline of the initial commit (`89f75c5`). Establishes the monorepo, the shared
contract, and a runnable process engine.

## Functional
- A process can be defined (tasks, service tasks, gateways, start/end) and **run
  end to end**: it pauses at human tasks and resumes on submit, calls connectors
  at service tasks, and routes at gateways.
- `pnpm demo` runs the onboarding flow in-process and prints the full trace
  (both gateway branches).

## Technical
- **pnpm monorepo**, run via `tsx` (no build step); workspace packages resolve
  `main → src/index.ts`.
- **`@cassiopeia/model`** — the shared contract: `ProcessDefinition`, `Node`
  (start/end/userTask/serviceTask/gateway), `Edge`, `Context`, plus lookup
  helpers. Imported by both web and api so canvas and engine can't disagree.
- **`@cassiopeia/engine`** — a **pure** engine (no I/O; connectors + audit
  injected via `EngineDeps`): `advance` / `completeTask`, token-based single
  active node, `validateDefinition`, event-sourced audit via `emit`.
- **`@cassiopeia/expr`** — a safe expression evaluator (recursive-descent, no
  `eval`) powering gateway conditions.
- **`apps/api`** — Fastify + `node:sqlite` (`DatabaseSync`, built into Node ≥ 22,
  zero native deps); the engine runs in-process. Vanilla SQL behind a thin repo
  so it can move to Postgres later.
- **`dev.mjs`** single launcher spawns api (:3001) + web (:5173) together.

## Verified
- `pnpm demo` prints a complete audit trail through both gateway branches.

# Milestone 12 — Agentic depth: tool-calling agents & HITL review

Shipped 2026-07-20

The differentiator. Agents that use tools while reasoning, humans who supervise
their decisions, guardrails on their output, and visibility into what they cost.

## Functional
- **Tool-calling agents**: an AI Agent connector can be given **tools** that map
  to other connectors (an API, an MCP tool, a mock). While reasoning, the model
  calls a tool, reads the result, and continues — up to a step cap — before
  producing its final answer.
- **Output guardrails**: declare **required output keys**; if the agent's JSON is
  missing any, it gets one corrective retry, and anything still missing is
  flagged (`guardrailMissing`) instead of silently passing.
- **Human-in-the-loop review**: review tasks now **pre-fill the form from the
  process context**, so a human sees the agent's proposed values (e.g. its
  `riskScore` / `decision` / `confidence`) and can approve or edit before the flow
  continues. Confidence-based routing works today via a gateway on the agent's
  `confidence` field.
- **Token & cost tracking**: every agent call records tokens and an estimated
  cost; the audit trail shows `agent.usage`, and **Stats surfaces total AI tokens
  and estimated cost** per workflow (also fed to the AI analyst).
- Configure tools + required keys in **Settings → the AI-agent connector**.

## Technical
- **connectors.ts** — `ai-agent` rewritten around a `chatCompletion()` helper and
  a **tool-calling loop** (OpenAI-style `tools` → `tool_calls` → run mapped
  connector → feed results back → repeat). Token usage accumulated across turns; a
  `PRICING` table estimates cost. Guardrail retry on missing `requiredKeys`.
  `runCtx` (`AsyncLocalStorage`) lets the adapter report usage without changing
  the engine's connector interface.
- **engine** — new `agent.usage` event type.
- **runtime.ts** — wraps each connector run in `runCtx` so `agent.usage` is
  emitted to the instance's audit log with `{model, tokens, cost, tools}`.
- **server.ts** — mock LLM now simulates a **tool round-trip** (returns
  `tool_calls` once, then a final decision with `confidence`) and reports `usage`,
  so the whole path is exercisable without credentials.
- **analytics.ts** — sums `agent.usage` into `aiTokens` / `aiCost`.
- **web** — Stats shows AI tokens + est. cost; Settings AI-agent editor gains a
  tools list + required-keys input; Portal/Inbox pass the instance context as
  `initial` to the form renderer (HITL pre-fill).

## Verified
- Live API (mock LLM): tool loop runs — output includes `toolUsed:true` from the
  tool result; a workflow instance completes with an `agent.usage` event
  (`450 tokens, ~$0.00081`); analytics report the same tokens/cost; the guardrail
  flags a missing required key (`guardrailMissing:["approvalCode"]`).
- Browser: Stats shows "AI tokens 450" + est. cost; Settings AI-agent tools +
  required-keys editor renders; app navigates all screens authenticated.

## Notes
- Real tool-calling quality depends on the configured model; verified end-to-end
  against the local mock. Set a real key in Settings to use a live model.

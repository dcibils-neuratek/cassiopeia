# Milestone 2 — Form runtime (form-kit) & portal

Baseline of the initial commit (`89f75c5`).

## Functional
- Human tasks render a **real form** to the end user, with field types (text,
  email, number, date, select, checkbox, file), **required/validation** rules,
  and **conditional visibility** (`visibleIf`).
- A **Portal** ("Run") lets you start an instance and complete its tasks as an
  end user, watching the instance context and audit trail update.

## Technical
- **`@cassiopeia/form-kit`** — a schema-driven React renderer (`FormRenderer`)
  with no form-library dependency: controlled inputs, `validate` / `coerce` /
  `initialValues` / `isVisible` helpers. The **same renderer** is used by the
  portal runtime and (later) the designer preview, so what you design is what
  runs.
- Form logic (`visibleIf`) reuses **`@cassiopeia/expr`** — the same evaluator as
  gateways, so one expression language spans routing and form visibility.
- Forms are part of the contract (`FormDefinition` / `FormField` in
  `@cassiopeia/model`); the api stores and serves them.

## Verified
- Conditional fields show/hide correctly and required validation blocks submit in
  the portal.

# Milestone 8 — Task Inbox & assignment

Shipped 2026-07-01 · commit `6870397`

Turn human tasks into a real worklist.

## Functional
- User tasks carry **assignee**, **candidate role**, and **priority**
  (low/normal/high), set in the designer alongside the SLA.
- A dedicated **Inbox** screen — a cross-instance worklist that:
  - lists every open human task with process, step, priority, assignee, and due
    date (overdue highlighted);
  - **filters** by workflow and "mine" (with a "You:" identity field);
  - lets you **claim** an unassigned task;
  - opens the task's form **inline** and **completes** it (or completes a
    no-form task) — the row drops off the list on completion;
  - sorts by **priority then soonest due**.

## Technical
- **model** — `UserTaskNode` gains `assignee`, `candidateRole`, `priority`.
- **db** — `tasks` gains `assignee` / `role` / `priority` columns (migration);
  `createTask` takes a `CreateTaskOpts`; `claimTask`; `rowToTask` helper.
- **runtime** — `onParked()` passes the node's assignment metadata into
  `createTask`.
- **server** — `GET /tasks` returns the enriched worklist (process/node/form
  names + instance context, sorted by priority then due); `POST /tasks/:id/claim`.
- **web** — new `Inbox.tsx` (worklist table + detail with claim, inline
  `FormRenderer`, instance data); Designer user-task assignee/role/priority/SLA
  fields; `App` nav entry + icon; identity persisted in `localStorage`.

## Verified
- Live API: enriched worklist (node/process/form names, priority, role, due) →
  claim sets the assignee → submit completes and merges form data → task removed
  from the open list.
- Browser: Inbox lists tasks (high before normal), Claim button, inline form
  loads; completing a task drops the worklist count 2 → 1.

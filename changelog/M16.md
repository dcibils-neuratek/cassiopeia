# Milestone 16 — Forms & documents

Shipped 2026-07-20

Richer forms for real-world intake: attachments, derived values, and multi-step
wizards.

## Functional
- **File upload with storage**: file fields now really **upload** — the file is
  stored server-side and referenced by id; the field value becomes
  `{ fileId, name, size, mime }`, and the file is retrievable at `/files/:id`.
  (Agents/connectors can be pointed at the stored file.)
- **Computed / derived fields**: a `computed` field shows a **read-only value
  derived from an expression** over the other fields (e.g. `price * quantity`),
  recomputed live and included in the submission.
- **Multi-page forms (wizard)**: assign fields to **pages** and the renderer turns
  the form into a stepper with **Back / Next**, validating each page before
  advancing.

> Repeating groups / dynamic panels remain future work.

## Technical
- **model** — `FieldKind` adds `computed`; `FormField` gains `expr` (computed) and
  `page` (wizard).
- **form-kit** — `computeDerived()` (uses `@cassiopeia/expr`), `formPages()`;
  `validate` skips computed fields; `visiblePatch` includes derived values.
  `FormRenderer` rewritten: page stepper (per-page validation), computed read-only
  control, and a `uploadFile` hook for real uploads.
- **db.ts** — `files` table (base64 content — fine for demo scale) +
  `saveFile` / `getFile`.
- **server.ts** — `POST /files` (base64 JSON, no multipart dep) and `GET
  /files/:id` (streams with the stored mime); Fastify `bodyLimit` raised to 20 MB.
- **web** — `uploadFile()` helper in `api.ts` (chunked base64); Portal + Inbox pass
  it to the renderer; FormDesigner exposes the `computed` kind + `expr` and a
  wizard-page field.

## Verified
- Live API: base64 upload of `note.txt` round-trips — `GET /files/:id` returns the
  original bytes.
- form-kit pure logic: `price*qty` computed field resolves to `30`; a 2-page form
  reports `pages [1,2]`; the submitted patch includes the derived `total`.
- Browser: app loads healthy and authenticated.

# Milestone 4 — Form designer

Baseline of the initial commit (`89f75c5`).

## Functional
- Select a **User Task** on the canvas and **design its form in a side drawer**
  without leaving the flow: add fields from a palette, edit per-field properties
  (label, bind, required, options, min/max, pattern, `visibleIf`), and **drag to
  reorder**.
- **Live preview** renders with the same runtime renderer the portal uses.

## Technical
- **`apps/web` FormDesigner** using **dnd-kit** for reordering; embeddable inside
  the Designer drawer (`fixedFormId` / `onClose`) or standalone.
- Preview shares **`@cassiopeia/form-kit`** — no divergence between design-time
  and run-time rendering.
- API: `/forms` + `/forms/:id` (GET/POST); forms stored as JSON.

## Verified
- Fields reorder via drag, properties persist, and the live preview matches the
  portal rendering.

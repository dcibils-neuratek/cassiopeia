# Milestone 11 — Identity, roles & secrets

Shipped 2026-07-20

Real users, real permissions, and secrets that never leave the server in the
clear. Prerequisite for any real pilot.

## Functional
- **Login** screen; every request now requires a session (except health, login,
  and the local mock endpoints). Sign out from the sidebar user chip.
- **Roles (RBAC)**, hierarchical: `viewer` → `operator` (run/inbox) → `analyst`
  (build/publish) → `admin` (settings/users). The UI hides what a role can't use
  and the API enforces it (403 otherwise).
- **Real task assignment**: the Inbox claims tasks for the logged-in user (no more
  free-text identity); an admin can claim on someone's behalf.
- **Audit log** — who did what (login, publish, start, claim, submit, retry,
  connector saves, restores, imports, user creates), viewable in Settings.
- **Admin console** in Settings: list users, **add users** with a role.
- **Connector secrets are encrypted at rest** and **never returned to the client**
  (masked); you can save other fields without re-entering a key.

## Technical
- **secrets.ts** — AES-256-GCM for secret config fields; key from
  `CASSIOPEIA_SECRET_KEY` or a generated `data/secret.key` (dev). Tolerates legacy
  plaintext; fails closed on undecryptable values.
- **db.ts** — `users` / `sessions` / `audit_log` tables; connector layer now
  `encrypt`s on save (with **merge-keep**: blank secret = keep stored),
  `decrypt`s on `getConnector` (execution), and `mask`s on `listConnectors` (UI).
- **auth.ts** — scrypt password hashing, opaque session tokens (7-day TTL),
  hierarchical `can(role, capability)`, `seedAuth()` (default `admin`/`admin`).
- **server.ts** — global `onRequest` auth hook; `requireCap()` guards on every
  mutating route; `addAudit()` on sensitive actions; endpoints `/auth/login`,
  `/logout`, `/me`, `/users` (GET/POST, admin), `/audit` (admin).
- **web** — `api()` attaches the bearer token and broadcasts `cass-unauth` on 401;
  `apiRaw()` for authenticated CSV download; `Login` screen; `App` auth gate +
  role-filtered nav + user chip/logout; Inbox uses the session user; Settings gets
  Users + Audit panels.

## Security decisions (defaults; hardening noted as follow-ups)
- Bearer token in `localStorage` (simple for the SPA) — XSS-hardening to an
  httpOnly cookie is a follow-up.
- Encryption key auto-generated in dev; **set `CASSIOPEIA_SECRET_KEY` in
  production** and keep it stable (rotating it makes existing secrets unreadable).
- Seed `admin`/`admin` on first run — change it in production.

## Verified
- Live API: unauth → 401; `admin`/`admin` login → token → 200; secret saved shows
  masked `""` to the client but `enc:…` at rest and **decrypts back** to the
  original for execution; merge-keep preserves the key when re-saving other
  fields; operator gets 200 on read/start but **403 on publish and connector
  writes**; audit log records the actor trail.
- Browser: login screen → authenticated app with user chip + logout, no console
  errors.

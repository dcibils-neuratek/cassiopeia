# Milestone 5 — Connectors & monitor

Baseline of the initial commit (`89f75c5`), plus the prompt-to-workflow builder
(`22efb8c`) and MCP connector (`0ea2657`).

## Functional
- **Service tasks call connectors**, configured and **tested inline** in the
  designer or in a Settings **connector library** (configure once, reuse):
  - **AI Agent** — provider-neutral OpenAI-compatible `/chat/completions`
    (`baseUrl` + `apiKey` + `model`); with `jsonOutput` the model's JSON merges
    into the process context so gateways can route on it.
  - **Maverick Agents** — native REST call to a Neuratek Maverick agent.
  - **MCP** — call a tool on an MCP server (Streamable HTTP); **Discover** lists
    the server's tools.
  - **HTTP** + deterministic mocks (`mock-verify`, `mock-mortgage`,
    `mock-credit-score`, `mock-travel-register`).
- **Monitor** shows every instance's status, context, and audit trail (live).
- **Platform UI**: Home, Stats (throughput chart), Templates (4 banking
  workflows), Build, Run, Settings; **✦ Describe** (LLM description) and
  **✦ Build with AI** (prompt-to-workflow chat).

## Technical
- **`apps/api/connectors.ts`** — adapters dispatch on connector `type`; secrets
  in config; `runConnector` injected into the engine. MCP does the JSON-RPC
  handshake (`initialize → tools/call`), handling session id and JSON/SSE.
- **describer** connector powers Describe + the AI builder (Claude Haiku default;
  any OpenAI-compatible provider). Local mock endpoints (`/mock-llm`,
  `/mock-maverick`, `/mock-mcp`) exercise every path without credentials.
- Neuratek-style design system via CSS variables.

## Verified
- Connectors testable from the UI; AI-agent path exercised end-to-end against the
  mock LLM; templates install and run on mock connectors.

# Milestone 3 — Process designer

Baseline of the initial commit (`89f75c5`).

## Functional
- A visual **canvas** (React Flow) to draw a process: add nodes from a palette,
  drag to connect, select nodes/edges to edit properties.
- **Gateway edges carry conditions** (and a default branch).
- **Validation** surfaces structural errors; **Publish** stores an immutable,
  versioned definition (draft = version 0; published = 1..N).

## Technical
- **`apps/web` Designer** on `@xyflow/react`: a custom `CassNode`, node/edge
  inspectors, and a `toDefinition()` serializer that maps canvas state to
  `ProcessDefinition` (gateways derive `branches` + `defaultEdgeId` from edges).
- **`validateDefinition`** (engine) is enforced at publish time so a broken flow
  can't be deployed; the designer shows the same errors inline.
- API: `/definitions/:id/edit|draft|publish`, versioning in `process_definitions`
  (PK `(id, version)`); the draft is kept in sync with the latest publish.

## Verified
- Publishing an invalid flow is rejected with actionable errors; a valid flow
  publishes and runs.

# Milestone 18 — UX & i18n polish

Shipped 2026-07-20

Make the studio pleasant and reach a wider audience.

## Functional
- **Dark mode**: a theme toggle in the top bar (persisted). The whole design
  system flips via a `data-theme="dark"` token set — the shell, cards, stats,
  settings, and the form renderer all adapt, with native controls/scrollbars
  following `color-scheme`.
- **i18n (Español / English)**: a language toggle in the top bar. The app shell
  (navigation, page titles, hints) is translated; strings fall back to English
  then the key, so deeper per-screen coverage can grow incrementally.
- **Canvas auto-layout ("⤢ Tidy")**: one click re-lays out the process
  left→right by BFS layer from the start node — instant cleanup of a messy or
  AI-generated diagram.

## Technical
- **index.css** — full `:root[data-theme="dark"]` token overrides (neutrals,
  brand, semantic tints, shadows) + `color-scheme`; the existing token-based
  components adapt with no per-component changes.
- **form-kit** — the renderer's styles moved onto CSS variables (with light
  fallbacks) so forms render correctly in either theme wherever they're embedded.
- **i18n.ts** — tiny `t()` + `getLang`/`setLang` with `en`/`es` dictionaries for
  the shell; `App` reads labels/titles/hints through `t()`.
- **App** — theme + language state (both persisted to `localStorage`), top-bar
  toggles.
- **Designer** — `autoLayout()` (BFS layering) behind the Tidy button.

> Remaining polish tracked as future: canvas undo/redo, global search, and
> extending i18n coverage into each screen.

## Verified
- Browser: toggling theme sets `data-theme="dark"` and the shell/cards/Home render
  dark; toggling language switches the nav to Spanish (Inicio / Estadísticas /
  Diseñar / Bandeja …); **Tidy** repositions all nodes on the canvas.

# Milestone 13 — Triggers & async

Shipped 2026-07-20

Start workflows from the outside, on a schedule, and let long-running work call
back when it's done.

## Functional
- **Webhook triggers**: create a trigger for a workflow (Build → Manage →
  **Automation**) and get a URL. Any system can `POST` JSON to it (no login — the
  token authorizes) to **start an instance**, with the posted body as the initial
  context.
- **Schedules**: start a workflow automatically on a **recurring interval**
  (e.g. every N minutes); managed in the same Automation tab, run by the
  background scheduler.
- **Async connectors**: an `async-callback` connector kicks off long-running
  external work and returns immediately; the instance **parks** until the external
  system `POST`s its result to a per-run **callback URL**, which resumes the flow
  with that data merged in.

## Technical
- **db.ts** — `triggers`, `schedules`, `callbacks` tables + CRUD (`createTrigger`
  / `getTrigger` / …, `createSchedule` / `dueSchedules` / `bumpSchedule`,
  `createCallback` / `getCallback` / `completeCallback`).
- **engine** — new `awaiting` `AdvanceResult`: a service task whose connector
  returns `__awaitCallback` parks (status `waiting`) with a callback token;
  `resumeCallback()` merges the callback data and advances. New events
  `service.pending` / `callback.received`.
- **connectors.ts** — `async-callback` adapter: generates a token, fires the
  kickoff (`{ input, callbackUrl }`), returns the await marker.
- **runtime.ts** — `startInstance(defId, initialContext?)`; `onParked` opens a
  callback row for `awaiting`; `resumeViaCallback(token, data)`;
  `fireDueSchedules()` wired into `startScheduler()` alongside timers.
- **server.ts** — public token routes `POST /hooks/:token` (start) and
  `POST /callbacks/:token` (resume); admin/build-guarded
  `/definitions/:id/triggers` and `/schedules` (GET/POST/DELETE).
- **web** — Designer Manage **Automation** tab (webhook URLs + schedules,
  create/delete); Settings `+ Async` connector with kickoff + callback-base config.

## Verified
- Live API: `POST /hooks/:token` (no auth) starts an instance whose context is the
  posted body (`{legalName, income}`); an async workflow parks
  (`service.pending`), then `POST /callbacks/:token` resumes it to `completed`
  with the callback data merged (`{approved, score}`), audit shows
  `service.pending → callback.received → instance.completed`; a 5s schedule
  started new instances (4 → 5) until deleted.
- Browser: Automation tab creates a webhook trigger (live URL) and shows the
  schedule form.
